## SRS (Software Requirements Specification)

Formalny dokument opisujący co system ma robić, bez szczegółów implementacyjnych.

### Cel
jednoznaczne zdefiniowanie wymagań
punkt odniesienia dla projektowania, implementacji i testów
podstawa do walidacji (czy system robi to, co miał robić)
### Typowa struktura (IEEE 830 / ISO)
1. Wprowadzenie: cel dokumentu, zakres systemu
2. Opis ogólny: kontekst systemu, użytkownicy (aktorzy)
3. Wymagania funkcjonalne: co system robi (często jako przypadki użycia)
4. Wymagania niefunkcjonalne: wydajność, bezpieczeństwo, dostępność, UX itd.
5. Interfejsy: API, UI, integracje
6. Ograniczenia: technologiczne, prawne, organizacyjne
